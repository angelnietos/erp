# Plan: Outlook Email Integration Module

## Overview
Integrate Microsoft Outlook email reading capabilities into the josanz-erp application to link email conversations with client data, enabling automatic enrichment of billing, delivery notes, and quotes information.

## Architecture

### Tech Stack
- **Frontend**: Angular (Nx workspace)
- **Backend**: NestJS (Node.js)
- **Database**: PostgreSQL with Prisma ORM
- **Authentication**: OAuth 2.0 via Azure AD / Microsoft Graph API
- **State Management**: Signals + Facade pattern (existing pattern)

### Module Structure
New module: `outlook-integration` spanning both backend and frontend layers.

## Phase 1: Backend Implementation (NestJS)

### 1.1 Database Schema Extensions
Add new tables to `prisma/schema.prisma`:

#### `OutlookAccount` - Stores connected Outlook accounts per tenant/user
```prisma
model OutlookAccount {
  id            String   @id @default(uuid())
  tenantId      String   @map("tenant_id")
  userId        String?  @map("user_id")  // optional - if per-user connection
  email         String   // outlook email address
  outlookId     String   @map("outlook_id")  // Microsoft Graph user id
  accessToken   String?  @map("access_token")
  refreshToken  String?  @map("refresh_token")
  tokenExpiresAt DateTime? @map("token_expires_at")
  scopes        String[]  // granted permissions
  isActive      Boolean  @default(true) @map("is_active")
  lastSyncAt    DateTime? @map("last_sync_at")
  createdAt     DateTime @default(now()) @map("created_at")
  updatedAt     DateTime @updatedAt @map("updated_at")
  
  tenant        Tenant   @relation(fields: [tenantId], references: [id])
  user          User?    @relation(fields: [userId], references: [id])
  
  @@map("outlook_accounts")
}
```

#### `EmailMessage` - Cached email messages
```prisma
model EmailMessage {
  id              String   @id @default(uuid())
  tenantId        String   @map("tenant_id")
  outlookAccountId String   @map("outlook_account_id")
  messageId       String   @map("message_id")  // Graph message id
  conversationId  String?  @map("conversation_id")
  subject         String
  bodyPreview     String?  @map("body_preview")
  bodyContent     String?  @map("body_content") @db.Text
  fromEmail       String?  @map("from_email")
  fromName        String?  @map("from_name")
  toRecipients    Json?    @map("to_recipients")
  ccRecipients    Json?    @map("cc_recipients")
  receivedAt      DateTime @map("received_at")
  isRead          Boolean  @default(false) @map("is_read")
  hasAttachments  Boolean  @default(false) @map("has_attachments")
  importance      String   @default("normal")
  categories      String[]
  isClientRelated Boolean  @default(false) @map("is_client_related")
  linkedClientId  String?  @map("linked_client_id")
  linkedEntity    String?  @map("linked_entity")  // "budget", "invoice", "delivery", "quote"
  linkedEntityId  String?  @map("linked_entity_id")
  syncedAt        DateTime @default(now()) @map("synced_at")
  createdAt       DateTime @default(now()) @map("created_at")
  
  outlookAccount  OutlookAccount @relation(fields: [outlookAccountId], references: [id], onDelete: Cascade)
  tenant          Tenant         @relation(fields: [tenantId], references: [id])
  client          Client?        @relation(fields: [linkedClientId], references: [id])
  
  @@map("email_messages")
}
```

#### `EmailConversation` - Thread grouping
```prisma
model EmailConversation {
  id              String   @id @default(uuid())
  tenantId        String   @map("tenant_id")
  outlookAccountId String   @map("outlook_account_id")
  conversationId  String   @map("conversation_id")  // Graph conversation id
  subject         String
  lastMessageAt   DateTime @map("last_message_at")
  messageCount    Int      @default(0) @map("message_count")
  participants    Json     // array of email addresses
  isResolved      Boolean  @default(false) @map("is_resolved")
  linkedClientId  String?  @map("linked_client_id")
  linkedEntity    String?  @map("linked_entity")
  linkedEntityId  String?  @map("linked_entity_id")
  createdAt       DateTime @default(now()) @map("created_at")
  updatedAt       DateTime @updatedAt @map("updated_at")
  
  outlookAccount  OutlookAccount @relation(fields: [outlookAccountId], references: [id], onDelete: Cascade)
  tenant          Tenant         @relation(fields: [tenantId], references: [id])
  client          Client?        @relation(fields: [linkedClientId], references: [id])
  messages        EmailMessage[]
  
  @@map("email_conversations")
}
```

### 1.3 Backend Services

#### `OutlookAuthService` - OAuth flow management
- Handles Azure AD app registration configuration
- Generates authorization URLs
- Exchanges authorization codes for tokens
- Refreshes expired access tokens
- Validates token scopes (Mail.Read, Mail.ReadWrite)

#### `OutlookSyncService` - Email synchronization
- Fetches emails via Microsoft Graph API
- Delta queries for incremental sync (/delta endpoint)
- Respects rate limits (10,000 requests/10 min)
- Implements exponential backoff for 429 responses
- Caches messages locally
- Updates conversation threads

#### `EmailLinkingService` - Smart email-to-client matching
- Analyzes email addresses against client database
- Matches by email domain patterns
- Extracts client references from email body/subject
- Links emails to existing budgets, invoices, delivery notes
- Creates activity logs on client records

#### `WebhookService` - Real-time notifications
- Subscribes to Graph API change notifications
- Handles webhook validation (validation token)
- Processes new email events in real-time
- Triggers sync jobs on new messages

### 1.4 API Controllers

#### `OutlookAuthController`
```typescript
GET  /api/outlook/auth/url          - Get authorization URL
GET  /api/outlook/auth/callback     - OAuth callback handler
POST /api/outlook/auth/connect      - Manual connection initiation
DELETE /api/outlook/auth/disconnect - Revoke access
GET  /api/outlook/auth/status       - Check connection status
```

#### `OutlookSyncController`
```typescript
POST /api/outlook/sync/start        - Trigger manual sync
POST /api/outlook/sync/stop         - Stop ongoing sync
GET  /api/outlook/sync/status       - Get sync status
GET  /api/outlook/sync/history      - Get sync history
```

#### `EmailMessagesController`
```typescript
GET  /api/outlook/messages          - List messages (paginated)
GET  /api/outlook/messages/:id      - Get message details
GET  /api/outlook/conversations     - List conversations
GET  /api/outlook/conversations/:id - Get conversation thread
POST /api/outlook/messages/:id/link - Link message to client/entity
DELETE /api/outlook/messages/:id/link - Remove link
```

### 1.5 Security Considerations
- **Token Storage**: Encrypt refresh tokens at rest using AES-256
- **Least Privilege**: Request only `Mail.Read` scope initially
- **Data Retention**: Auto-delete emails older than 90 days (configurable)
- **Audit Logging**: Log all sync operations and data access
- **Rate Limiting**: Per-tenant sync frequency limits
- **PII Handling**: Mask sensitive data in logs

## Phase 3: Business Logic & Automation

### 3.1 Smart Linking Rules
1. **Exact Email Match**: Client.email === email.from
2. **Domain Match**: client.contacts[].email domain matches
3. **Subject Keywords**: Extract client ID/reference from subject
4. **Body Pattern Matching**: Regex for client codes
5. **Reply Chain**: Link to same client as parent email

### 3.2 Automated Actions
- New email from client → Create activity log
- Invoice mentioned in email → Flag for review
- Delivery confirmation email → Auto-mark delivery complete
- Contract/po references → Link to budget

### 3.3 Reporting
- Emails per client report
- Response time metrics
- Email-to-invoice conversion
- Unlinked email alerts

## Phase 4: Configuration & Deployment

### 4.1 Azure AD App Registration Required
```
Permissions:
- Microsoft Graph > Delegated:
  - Mail.Read
  - Mail.ReadWrite (optional)
  - User.Read
  - offline_access

Redirect URIs:
- https://{app-domain}/api/outlook/auth/callback
```

### 4.2 Environment Variables
```bash
# Backend
OUTLOOK_CLIENT_ID=azure-app-client-id
OUTLOOK_CLIENT_SECRET=azure-app-secret
OUTLOOK_TENANT_ID=azure-tenant-id
OUTLOOK_REDIRECT_URI=/api/outlook/auth/callback
OUTLOOK_SCOPES=Mail.Read offline_access
EMAIL_SYNC_CRON=0 */2 * * *  # Every 2 hours
EMAIL_RETENTION_DAYS=90

# Frontend
OUTLOOK_AUTH_BASE_URL=/api/outlook
```

### 4.3 Docker Considerations
- No outbound port changes needed (HTTPS to graph.microsoft.com)
- Environment variable injection via secrets
- Token encryption key via Kubernetes secret

## Dependencies & Risks

### Dependencies
1. Azure AD tenant and app registration
2. Microsoft Graph API availability
3. User consent for permissions
4. Existing client email data quality

### Risks & Mitigations
1. **Rate Limiting**: Implement queue + exponential backoff
2. **Token Expiration**: Proactive refresh before expiry
3. **Data Privacy**: Encryption at rest, audit trails
4. **Sync Performance**: Delta queries, pagination, background jobs
5. **Matching Accuracy**: Manual override always available
